# Task 02: Evolução de Branding e Elementos Visuais do Repositório

## Contexto
Atualmente, o repositório possui o nome genérico "TypeScript Boilerplate". Para ganhar relevância de mercado ("GitHub Stars"), ele precisa se posicionar não apenas como um projeto, mas como um "Produto" ou "Framework". Além disso, a principal feature (gerador CLI de domínio) não possui representação visual no README.

## Objetivos
- Dar identidade própria ao boilerplate.
- Criar impacto visual imediato ("Mind Blown") para os desenvolvedores que abrem o README.

## Itens de Ação
- [x] Definir um nome fantasia ("Praxis") como branding secundário no repositório.
- [x] Ajustar os fluxos interativos dos `.tape` (ex: `bun db:migrate:push` com 'Yes').
- [x] Refinar o visual da CLI (`bun gen:domain`) garantindo logs premium (cores, limpeza) antes da gravação final.
- [x] Gravar os GIFs finais via VHS (`praxis-content-create-demo.tape`, `praxis-test-speed.tape`).
- [x] Criar e gravar teste de carga (brute force) demonstrando resiliência e performance (`praxis-load-test.tape`).
- [x] Integrar os GIFs gerados no README de forma a criar impacto imediato ("Mind Blown").
- [x] Percorrer a pasta `/docs` e integrar os GIFs em locais estratégicos (Getting Started, Database, Testing, etc).
- [ ] Considerar extrair a CLI `bun gen:domain` para um pacote NPM isolado no futuro.

## Resultados Esperados
- Aumento do engajamento orgânico no GitHub.
- Maior tempo de retenção na leitura do repositório devido ao apelo visual (GIF/Vídeo).
