## 2024-05-18 - [Cache Infrastructure N+1 and Key Bug]
| Learning: The `get` function for cache was executing sequential asynchronous iterations (`await client.get(hash)`) inside a `for` loop for every matched pattern from `scan`, which creates an extreme N+1 bottleneck. Furthermore, it incorrectly used the original `hash` parameter instead of the matched `keys[i]` on each iteration, thus repeatedly fetching the same base key instead of properly retrieving all matched pattern keys.
| Action: Refactored `get` and `del` in `src/infrastructure/cache/actions.ts` to correctly map the matched `keys` array and execute the redis calls concurrently using `Promise.all()`. Tests were updated to assert `get` correctly returns `{ key1: value, key2: value }` instead of `{ originalHash: value }`.
