## 2024-05-24 - [Rate Limiting Bypass] | Vulnerability: The Fastify webserver lacked the `trustProxy: true` configuration. This could allow users behind a proxy or load balancer to bypass IP-based rate limiting or conversely, rate limit the proxy IP itself, causing a Denial of Service for all users behind it. | Learning: Always configure `trustProxy: true` when relying on client IP addresses (e.g. for rate limiting) in an environment that may be deployed behind a proxy or load balancer. | Prevention: Enforce proxy awareness for any framework utilizing `request.ip` for security controls.
