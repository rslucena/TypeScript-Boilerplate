## 2026-03-16 - Proxy Trust Misconfiguration | Vulnerability: Unconditionally trusting the proxy using `trustProxy: true` opens the application up to IP spoofing where malicious clients can forge their X-Forwarded-For headers. | Learning: When configuring Fastify behind a reverse proxy, the `trustProxy` option must be carefully controlled, ideally through configuration (e.g., env vars), allowing either specific IP ranges, strict booleans only in highly controlled environments, or completely off by default. | Prevention: Introduce an `APP_TRUST_PROXY` environment variable (default: false) mapped to Fastify's configuration to explicitly define which proxies to trust, if any.
